00333 /* 
00334  * OSPF Fragmentation / fragmented writes
00335  *
00336  * ospfd can support writing fragmented packets, for cases where
00337  * kernel will not fragment IP_HDRINCL and/or multicast destined
00338  * packets (ie TTBOMK all kernels, BSD, SunOS, Linux). However,
00339  * SunOS, probably BSD too, clobber the user supplied IP ID and IP
00340  * flags fields, hence user-space fragmentation will not work.

00343  * supports it, and which avoids nasty IP ID state problems.
